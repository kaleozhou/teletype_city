# .gitignore 文件说明

## 📁 文件概述

这个 `.gitignore` 文件专门为《终端·回响》项目设计，确保不会将不必要的文件提交到Git仓库中。

## 🚫 忽略的文件类型

### Python 相关
- `__pycache__/` - Python字节码缓存目录
- `*.py[cod]` - Python编译文件
- `*.so` - 编译的扩展模块
- `build/`, `dist/` - 构建和分发目录
- `*.egg-info/` - Python包信息

### 虚拟环境
- `venv/` - 虚拟环境目录
- `env/`, `ENV/` - 其他虚拟环境目录
- `.venv/` - 隐藏的虚拟环境目录

### IDE 和编辑器
- `.vscode/` - Visual Studio Code 配置
- `.idea/` - PyCharm/IntelliJ 配置
- `*.swp`, `*.swo` - Vim 临时文件
- `*~` - 编辑器备份文件

### 系统文件
- `.DS_Store` - macOS 系统文件
- `Thumbs.db` - Windows 缩略图缓存

### 日志文件
- `*.log` - 所有日志文件
- `logs/` - 日志目录
- `game_server.log` - 游戏服务器日志

### 游戏数据和备份
- `backups/` - 备份目录
- `data/players.json` - 玩家数据（包含敏感信息）
- `data/world_state.json` - 世界状态数据
- `data/game_stats.json` - 游戏统计数据

### Docker 相关
- `docker-compose.override.yml` - Docker Compose 覆盖文件
- `*.env` - 环境变量文件

### 测试相关
- `.pytest_cache/` - pytest 缓存
- `.coverage` - 代码覆盖率数据
- `htmlcov/` - HTML 覆盖率报告

### 临时文件
- `*.tmp` - 临时文件
- `*.bak` - 备份文件
- `temp/`, `tmp/` - 临时目录

## 🔧 自定义配置

### 添加新的忽略规则
如果您需要忽略其他类型的文件，可以在相应的分类下添加：

```bash
# 自定义规则
*.custom
custom_folder/
```

### 强制包含文件
如果某个被忽略的文件需要强制包含，可以使用 `!` 前缀：

```bash
# 忽略所有 .log 文件
*.log

# 但包含重要的日志文件
!important.log
```

## 📋 常见使用场景

### 开发环境
```bash
# 克隆项目后，虚拟环境会被忽略
git clone <repository>
cd teletype_city
python3 -m venv venv  # 这个目录不会被提交
source venv/bin/activate
```

### 数据管理
```bash
# 游戏数据不会被提交，保护玩家隐私
# 但配置文件会被提交
git add data/rooms.yml    # ✅ 会被提交
git add data/players.json # ❌ 会被忽略
```

### 部署配置
```bash
# 本地环境配置不会被提交
cp .env.example .env      # 创建本地配置
# .env 文件会被忽略，不会提交到仓库
```

## ⚠️ 注意事项

1. **敏感信息**：确保包含敏感信息的文件（如数据库、密钥）被忽略
2. **大文件**：避免提交大文件，使用 Git LFS 或外部存储
3. **临时文件**：定期清理临时文件和缓存
4. **环境特定**：不同环境的配置文件应该被忽略

## 🚀 最佳实践

1. **定期检查**：使用 `git status` 检查是否有意外文件
2. **团队协作**：确保团队成员都了解 .gitignore 规则
3. **文档更新**：当添加新的忽略规则时，更新此文档
4. **测试验证**：在提交前验证 .gitignore 是否按预期工作

## 🔍 验证 .gitignore

```bash
# 检查哪些文件会被忽略
git status --ignored

# 检查特定文件是否被忽略
git check-ignore filename

# 查看所有被忽略的文件
git ls-files --others --ignored --exclude-standard
```
